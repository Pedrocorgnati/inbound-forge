# UX/UI pass — 2026-06-10

Auditoria UX/UI multi-agente (7 dimensoes) + correcoes. As 5 features de inbound
(F1-F5) tinham sido construidas com HTML cru; esta passada as alinhou ao design
system e corrigiu IA, estados e acessibilidade.

## Causa-raiz (resolvia ~10 achados HIGH de uma vez)
As paginas/componentes das 5 features usavam `<button>/<input>/<select>/<textarea>/
<p>` crus em vez do component-kit. Refatorados para o kit, o que corrige de uma vez:
consistencia visual, a11y (labels via Input/Select, focus-visible, touch-target 44px),
mobile (sem zoom-on-focus iOS), forms-ux (spinner isLoading), estados e dark mode.

## O que foi feito
1. **7 componentes -> component-kit** (Button/Input/Select/Textarea/Checkbox):
   SubscribeFormClient, LeadFormRenderClient (publicos, conversao), BroadcastComposer,
   FormBuilder, SequenceBuilder, AutomationBuilder, BroadcastSendButton.
   - Botoes raw -> `<Button isLoading loadingText>` (spinner + min-h-44px + focus ring).
   - Inputs/selects/textareas raw -> `<Input/Select/Textarea label>` (label associado = a11y).
   - Checkbox raw -> `<Checkbox label onCheckedChange>` (Radix, acessivel).
   - `window.confirm/alert` (BroadcastSendButton) -> `<ConfirmDialog>` + `toast`.
   - **Zero Silencio**: `toast.success/error` em todo save (antes nao havia feedback).
   - Cores hardcoded (`text-red-600`, `bg-green-50`) -> tokens semanticos
     (`text-danger`, `success-bg/text`) que respeitam dark mode.
2. **5 list pages**: empty/error `<p>` cru -> `<EmptyState variant>` (com CTA em broadcasts);
   status como texto -> `<Badge variant>` (mapeado por status). `<th scope="col">` (a11y).
3. **Estados scoped**: `loading.tsx` (RouteLoadingState) + `error.tsx` (RouteErrorState)
   para broadcasts/subscribers/forms/sequences/automations (antes: erro = takeover full-page).
4. **IA da navegacao**: sidebar de 23 itens flat -> 6 secoes (Visao geral, Conteudo,
   Publicacao, Email & Automacao, Leads & CRM, Analise & Sistema). `NAV_SECTIONS` +
   `NAV_ITEMS` derivado (flatMap) p/ compat; sidebar-nav renderiza headings (some no modo
   colapsado, com Separator entre secoes). Label ambigua 'Email' -> 'Broadcasts'.
5. **i18n** 4 locales (755 chaves, paridade): nav.sections.*, builder.cancel/saved.
